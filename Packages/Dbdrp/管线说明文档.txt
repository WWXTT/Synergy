  核心区别总结

  Danbaidong RP 在 URP 基础上做了三个方向的增强：

  1. 卡通渲染 — PBR+NPR 融合的 Toon Shader 系列、自定义 Bloom、描边系统、角色专属光照 Pass
  2. 光追 — 把 HDRP 才有的 Ray Tracing 能力移植到了 URP（精简版，不含降噪器）
  3. 阴影增强 — 逐对象阴影 + 屏幕空间阴影，突破 URP 原生阴影的局限

整体渲染流程      
核心思路是 "延迟写深度，前向着色"（Deferred Depth + Forward Shading），这是一种半延迟（Deferred+Forward 混合）架构。                                                              
按 UniversalRendererRenderGraph.cs 的执行顺序：
  1. GBuffer Pass      → 只写 深度 + 法线 到 GBuffer（不做光照计算）
  2. DeferredLighting  → 对 GBuffer 做延迟光照（场景静态物体等）
  3. CharacterForward  → 角色前向着色（利用 GBuffer 已有的深度做 ZTest Equal）
  4. OpaqueForwardOnly → 其他需要前向渲染的物体
  5. Skybox
  6. Transparents      → 透明物体前向渲染

角色不透明物体：GBuffer 只当 Depth Prepass 用
  关键：角色前向 pass 设置了 ZWrite Off, ZTest Equal。因为 GBuffer pass 已经写入了正确的深度，CharacterForward pass
  只需要在相同像素位置做前向着色，不会产生深度冲突，也不会被延迟光照的结果覆盖。

透明物体：纯前向渲染
  PBRToonTrans.shader 完全不走 GBuffer：
  Queue = Transparent
  两个 pass：
  - TransparentBackface（LightMode="TransparentBackface"）— Cull Front，先渲染背面
  - UniversalForward（LightMode="UniversalForward"）— 正常渲染正面

各向异性（头发高光）：在前向 pass 中计算
  PBRToonHair.shader 的各向异性高光不是 Kajiya-Kay 或 d'Eon 等基于切线的方法，而是一种 基于视空间 Y 分量的纹理查找 +
  Blinn-Phong 混合方案：
  - _AnisotropicSlide 控制视角 Y 方向对纹理采样位置的偏移量（模拟各向异性的方向性）
  - _AnisotropicOffset 整体偏移
  - _HairSpecTex 是一张各向异性高光遮罩纹理
  - 最终与标准 GGX 高光叠加：specular = GGX spec + hairSpecTerm

  这种方式完全在 CharacterForward pass 中完成，不需要 GBuffer
  存储额外的各向异性数据，所以在延迟渲染路径下不需要特殊处理。

  总结
  ├──────────────────────────┼───────────────────────────────────────────────────────────────────────────┤
  │ 卡通风光照无法走标准延迟 │ GBuffer pass 只写深度/法线，真正的着色在自研 CharacterForward pass 中完成 │
  ├──────────────────────────┼───────────────────────────────────────────────────────────────────────────┤
  │ 透明物体                 │ 纯前向渲染，TransparentBackface + UniversalForward，不走 GBuffer          │
  ├──────────────────────────┼───────────────────────────────────────────────────────────────────────────┤
  │ 各向异性高光             │ 在前向 pass 中用视空间偏移采样 + Blinn-Phong 计算，不需要 GBuffer 扩展    │
  ├──────────────────────────┼───────────────────────────────────────────────────────────────────────────┤
  │ 描边                     │ 自研 CharacterOutline LightMode，在 CharacterForward pass 中一并渲染      │
  └──────────────────────────┴───────────────────────────────────────────────────────────────────────────┘
PBRToon 系列虽然最终走前向着色，但 GBuffer pass 仍然在写深度和法线。 如果是写实风格 PBR，写完整的 GBuffer pass就能享受延迟渲染的好处。如果是卡通风或需要特殊光照模型，就参照 PBRToonBase 的模式：GBuffer    只写深度+法线，着色放在 CharacterForward 里做。

🎨 卡通渲染管线（已实现）
PBR Toon Shader & Character Forward Lighting
项目的核心模块。在 URP 的基础上实现了 PBR（基于物理）与 NPR（非物理）融合的卡通渲染着色器，并提供了针对角色优化的特殊前向渲染路径（Inserted GBuffer Passes）。这使得角色可以在 GBuffer 阶段插入自定义数据，为后续的卡通光照计算提供必要信息，兼顾了风格化表现与物理光照的融合。

Danbaidong Shader GUI
自定义着色器编辑器面板。使用方式是在自己的 Shader 中添加 CustomEditor "UnityEditor.DanbaidongGUI.DanbaidongGUI"，即可获得专用于卡通渲染参数调节的可视化界面，简化了 Toon Shader 的调试与配置流程。

Toon Bloom
针对卡通风格优化的泛光后处理效果，使高光溢出更符合动画/漫画的视觉风格，而非写实管线的物理泛光。

Anime Toonmapping
动漫风格的色调映射，将 HDR 颜色映射为符合动画美术风格的 LDR 输出，区别于传统影视级 Tone Mapping 追求真实感的策略。

SmoothNormal 描边支持
通过独立的 SmoothNormal 包（https://github.com/danbaidong1111/SmoothNormal.git）提供平滑法线数据，只需将模型文件名加上 _SN 后缀即可使用，用于实现无棱角断裂的描边效果。

🔦 阴影系统（已实现）
PerObjectShadow（逐对象阴影）
URP 原生只提供全局统一的阴影设置，而 DanbaidongRP 实现了逐对象独立阴影控制。这对主角或关键物体的阴影精度表现和性能优化非常有价值——可以为重要角色分配更高精度的阴影资源，同时降低次要物体的阴影开销。使用方式：添加 Renderer Feature PerObjectShadowFeature，并在目标 Mesh 上挂载 PerObjectShadowProjector 脚本。

ScreenSpaceShadow（屏幕空间阴影）
针对定向光的屏幕空间阴影效果，作为 Renderer Feature 集成，使用时需添加 ScreenSpaceShadows Feature。这是一种在屏幕空间中解析阴影的技术，可以提供更柔和的阴影边缘表现。

🛠️ 工具链与开发体验（已实现）
Wizard for Default Presets（管线创建向导）
完成管线资产创建和 Graphics 设置后重启项目，会自动弹出一个图形化向导窗口，引导用户完成 DanbaidongRP 所需的必要配置，降低了初始设置的门槛和出错概率。

多包 UPM 分发架构
项目拆分为 Core、Config、DanbaidongRP 三个独立包，通过 Unity Package Manager 以 Git URL 方式安装，结构清晰，便于版本管理和增量更新。

📌 小结
DanbaidongRP 当前的定位不是一个试图将 HDRP 全部特性搬到 URP 上的通用高保真管线，而是一个以卡通/动画风格渲染为核心、在 URP 基础上定向增强特定能力的自定义管线。它的已完成模块（Toon Shader、Toon Bloom、Anime Toonmapping、PerObjectShadow、Inserted GBuffer）都服务于同一个目标：在 URP 的性能基础上，为卡通风格项目提供更高品质的渲染表现。

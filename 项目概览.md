# Synergy - TCG/CCG 卡牌游戏规则引擎

双人卡牌对战游戏（TCG/CCG）的规则引擎框架，融合《游戏王》《万智牌》《炉石传说》三款游戏的核心机制，统一为六元素体系（灰/红/蓝/绿/白/黑）。

## 代码风格
- PascalCase 命名类和方法，camelCase 命名私有变量
- 枚举使用英文命名，中文仅用于注释和显示

## 核心架构
```
GameCore (顶层调度器)
├── TurnEngine           - 回合阶段状态机
├── EffectExecutionEngine - 效果执行 + 栈引擎 + 触发引擎
├── LayerEngine          - 层系统（持续效果计算）
├── StateBasedActions    - 状态动作检查
├── ReplacementEngine    - 替代效果引擎
├── ZoneManager          - 区域管理器 (Zones.cs)
├── ElementPool          - 元素池系统（法力资源）
├── ElementUnlockManager - 元素解锁管理（白/黑需解锁）
├── CombatSystem         - 战斗系统
├── SummonEngine         - 额外卡组召唤（融合/同调/超量/连接）
├── AttributeSystem      - 属性修正系统
├── TimestampSystem      - 时间戳管理
└── EventManager         - 统一事件总线
```

## 六元素体系

| 颜色 | 英文 | 特性 | 对应 |
|------|------|------|------|
| 灰色 | Gray | 通用/检索 | 无色 |
| 红色 | Red | 伤害/破坏/敏捷 | 火焰、攻击性 |
| 蓝色 | Blue | 控制/知识/抽卡 | 操控、法术 |
| 绿色 | Green | 成长/召唤/治疗 | 自然、恢复 |
| 白色 | White | 秩序/保护 | 光（需解锁） |
| 黑色 | Black | 牺牲/操控 | 暗（需解锁） |

## 已实现系统

### 战斗系统 (CombatSystem)
完整战斗流程状态机：攻击宣言 → 阻挡宣言 → 伤害计算 → 伤害结算

### 连锁/栈系统 (StackEngine)
- LIFO 栈结算
- 速度计数器（SpeedCounter）控制连锁深度
- 优先权传递

### 触发系统 (TriggerEngine)
- 88 种触发时机
- 自动/强制/自由发动类型
- 事件匹配与自动入栈

### 层系统 (LayerEngine)
5 层持续效果应用：控制权 → 复制 → 文本 → 能力修改 → 属性修改
含 CDA（特征定义能力）和持续效果时长追踪

### 召唤系统 (SummonEngine)
融合/同调/超量/连接四种额外卡组召唤方式，每种有独特的材料验证规则

### 区域系统 (ZoneManager)
手牌/战场/坟墓场/流放区/牌库/元素池/额外卡组/场地区/灵摆区

### 事件系统 (EventManager)
统一事件总线，支持广播和定向两种模式，含对象池和 Unity 对象有效性检查
50+ 种游戏事件覆盖回合/卡牌/战斗/效果/状态全场景

### 元素池系统 (ElementPool)
横置手牌作为元素源，按颜色管理可用资源，支持元素解锁机制

### 价值系统 (ValueSystem)
卡牌价值评估框架，协同加成计算，用于平衡性调整

### 状态动作系统 (SBA)
自动检测并处理：生命归零、防御力归零、传奇规则等

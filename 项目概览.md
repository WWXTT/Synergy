# Synergy - TCG/CCG 卡牌游戏规则引擎

双人卡牌对战游戏（TCG/CCG）的规则引擎框架，融合《游戏王》《万智牌》《炉石传说》三款游戏的核心机制。

## 游戏规则

### 区域分布
- 游戏以正六变形铺满场地，每个玩家分别具有 额外1格 卡组1格 墓地1格 除外区1格 生物召唤区域6格 非生物使用区域6格 元素池6格  
- 区域功能 额外放置最多15张额外怪兽，基于游戏王的特招规则见后续

### 费用体系 （代价系统）
- 红蓝绿灰四色元素是最基本的代价，卡牌的使用，效果的发动都需要支付代价。
- 元素是每回合准备阶段，玩家从手牌中挑选一张放进元素池，基于该卡的费用构成，放置对应数量的元素指示物，一回合一次，去除一个指示物，玩家获得对应元素，指示物消耗完毕后会直接进入墓地
- 红蓝绿划分效果集中功能倾向明确的部分，灰色是难以分类的，或通用的效果。黑白是预留拓展色，暂时不制作

### 特招规则
一、融合召唤
核心条件：
关键词要求：必须使用至少一个带关键词的生物作为素材 
召唤方式：牺牲符合条件的素材生物，如果牺牲的费用小于要特招的随从，则支付剩余法力费用
独特优势：融合生物继承素材的关键词，重复可叠加关键词。比如：关键词，圣盾，抵挡一次伤害，融合生物本身具有圣盾，又融合了一个圣盾素材，此后它的具有的圣盾将抵挡两次伤害。

二、同步召唤
核心条件：
调整要求：必须使用至少一个"调整"生物，和场上至少一个非调整生物
费用门槛：作为素材的非调整生物的总费用必须大于同步怪兽的费用
时机灵活：调整生物可通过额外支付费用，提高自身效果发动速度，进行同步召唤时，将自身的效果发动速度赋予该同步怪兽。
独特优势：同步的召唤和效果，发动速度快，对手难以连锁响应

三，超量召唤
核心条件：
超量怪兽没有法力值，只能配置发动时扣除法力值的效果
素材叠放：将场上任意数量的生物作为超量素材置于超量怪兽下方特殊召唤
动态属性，攻击力/生命值 由超量素材的对应属性值总和决定
独特优势：发动效果时，可选择支付法力费用或拔除对应费用的超量素材替代，可将多个小生物转换为大生物，对于基于生物属性计算的效果具有优势


四、链接召唤
核心条件：
链接怪兽必须具有增益和减益效果
箭头要求：必须使用至少一个带方向箭头的生物作为素材
费用门槛：素材生物的总费用必须大于链接生物的费用
箭头继承：链接生物继承所有素材的箭头方向
独特优势：通过链接网络将增益和减益效果作用到更多的格子，相互链接时，共享效果

注：额外怪兽不能作为额外怪兽的素材 [关键词]一种仅作用于自身的原子效果 [调整][方向箭头]均为生物标签的一种 一个生物可用具有多个标签

### 原子效果
所有卡牌效果均遵循单一数据流管道：
触发 ➔ 条件过滤 ➔ 支付代价 ➔ 锁定目标 ➔ 执行动作

1. 发动时机与条件 —— 事件驱动机制和速度系统
时机分为系统事件和原子效果事件，阶段上又分为发动，开始作用，结算完成。增加新的原子效果，就对应自动增加该效果发动的三个事件

2.作用目标与范围 —— 筛选与策略分离
目标选择 = 筛选器 + 决策权。
筛选器定义合法对象（如：敌方、野兽、攻击力>5），生成弹出展示筛选出的对象，玩家自行选择最终目标

3. 原子效果组装 —— 原子效果与链式调用
在炉石和万智牌的关键词效果中，挑选9个效果，红蓝绿各三个，硬编码作为测试数据
原子效果采用节点连接方式组装。复杂效果通过“条件分支节点”串联（例如：造成伤害 -> 判断是否死亡 -> 是则抽卡）

4.费用体系统一 —— 代价泛化，代价需要像原子效果一样可以拓展
Summon 结合游戏王特色的额外卡组特殊召唤的条件，也需要纳入代价系统
目前暂定四种，元素消耗，弃牌，扣除玩家生命，沉睡（卡翻面且增加一个苏醒倒计时），等都视为“支付代价”。发动卡牌前，需通过支付。

- ###  栈/连锁系统
- 卡的效果发动分两种，条件发动即基于事件的发动和速度发动。
- 速度发动是所有的玩家自主决定发动时机的卡和效果，都具有一个默认的发动速度，主要阶段，回合持有者1，非回合持有者0，非主要阶段，双方都是0。
- 场上有一个记速器，默认值0，双方玩家需要发动速度超过这个值，才能发动，然后入栈的同时记速器+1。双方轮询至没有人发动更高速度的效果开始结算。
结算完成一个记速器-1，但是开始结算后，不再轮询，一直到本次结算完成，记速器归0，才能进行下一次双方轮询发动效果。
-速度值来源，卡可以在编辑阶段通过添加速度相关的效果，来提高自己的发动速度，协调怪兽可以在对局中消耗额外的费用来动态提高自己的发动速度，
且协调的速度继承给用它作为素材的同步怪兽。

###
总结：这是一个基于颜色效果分类的，将原子效果组装成效果，将效果组装成卡，用卡组装成卡组，最后像普通卡牌对战游戏一样游玩的联机卡牌游戏。

## 项目架构

### 技术栈
- Unity 6000.5.0a8 / .NET Standard 2.1 / C# 9.0
- Mirror 网络框架（Telepathy/kcp2k 传输层）
- MemoryPack 1.21.4（二进制序列化，网络消息传输）
- DOTween / UniTask

### 核心目录结构
```
Assets/Core/
  Attribute/         # 原子效果系统 + Handlers
  BaseSystems/       # 核心游戏系统（TurnEngine, CombatSystem, ElementPool 等）
  Cards/             # 卡牌数据模型（CardData, Card, CardWrapper, CardTypes 接口）
  Effects/           # 效果系统（EffectDefinition, SpeedSystem, TriggerTiming, ReplacementEngine）
  Enums/             # 枚举定义（CardEnums, ColorIdentity, Zones）
  Events/            # 事件系统（EventManager, GameEvent）
  GameCore/          # 游戏核心（GameCore 门面, GameState, SubSystemRegistry）
  Layers/            # 层系统（LayerEngine, ControlChangeLayer, TextChangeLayer）
  Network/           # 网络消息层（NetworkMessage 信封, GameMessages DTO, NetworkSerializer）
  Serialization/     # 序列化层（MemoryPack DTO, TagTable, EffectSerializer）
  Summon/            # 召唤引擎
  Utility/           # 工具类（MurmurHash3）
  ValueSystem/       # 配置（SynergyConfig）
  Synty/			 # 美术资源集，AI可忽略该文件夹，游戏表现不在你的工作范围内
```

### 序列化架构
**双轨制**：Unity Inspector 用 `[Serializable]`/`[SerializeField]`，网络传输用 MemoryPack DTO。

| 层 | Unity 序列化 | MemoryPack 序列化 |
|---|---|---|
| 数据类 | `CardData`, `EffectData` | `SerializableCardData`, `SerializableEffectData` |
| 运行时状态 | `Card._power` 等 internal 字段 | `SerializableRuntimeCardState` |
| 效果定义 | `EffectDefinition`, `AtomicEffectInstance` | `SerializableEffectDefinition`, `SerializableAtomicEffectInstance` |
| 网络消息 | — | `NetworkMessage` 信封 + `MsgPlayCard` 等 |

**MemoryPack 标签系统**：
- 属性标签：`TagTable.cs` 中的 `const int`，由 `Tools/刷新MemoryPackOrder` 编辑器工具从 `"ClassName.PropertyName"` 经 MurmurHash3 自动生成
- 效果标签：64 位 MurmurHash3，从效果描述文本动态生成
- 新增属性只需在编辑器工具的 `GetTagDefinitions()` 加一行，点菜单刷新即可

**关键转换路径**：
```
JSON → CardLoader → CardData → SerializableCardData → byte[] (网络传输)
byte[] → SerializableCardData → CardData → CardWrapper (运行时)
Card → SerializableRuntimeCardState → byte[] (状态同步)
```

### 卡牌数据模型
- `CardData`：纯数据容器（ID, Supertype, CardName, Life, Power, Cost, Effects, Keywords, Tags）
- `Card`：partial class，继承 Entity，通过接口组合属性（IHasSupertype, IHasName, IHasLife 等）
- `CardWrapper`：连接 CardData 和 Card，实现所有接口
- Entity 效果扩展字段在 `EntityEffectExtensions.cs` 的 partial Card 中

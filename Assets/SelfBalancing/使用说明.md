# LIPM 自平衡 Active Ragdoll —— 使用说明

## 〇、一句话心智模型

```
屏幕上看到的角色 = Ghost_Rig（唯一渲染的骨架）
  第一层  基础动画      Animator 时间线          —— 无参数，底座
  第二层  融合动画      MagicBlend globalWeight  —— 一个独立参数（动画侧）
  第三层  物理动画      PhysicsPoseBlender        —— 一个独立参数 balanceUrgency（物理侧）

Physical_Rig  = 不渲染，纯物理传感层，向第三层提供「倾倒 / COM 偏移 / 单摆方向」
Shadow_Mimic  = 不渲染，纯碰撞替身，跟随 Ghost 可见姿态，供对外受击/推物
```

**两个参数互不从属**：调 `globalWeight` 不影响物理层，调 `balanceUrgency` 不影响动画融合层。

---

## 一、生成一个角色

1. 准备一个带 **Humanoid Animator**（已配 Avatar）的角色模型，摆好 AnimatorController（至少有一个默认 idle/locomotion 状态）。
2. 在该模型根物体上挂 `ActiveRagdollBrain`，在 Inspector 指定一个 `RagdollTuningProfile`（可用 `CharacterProfiles/DefaultProfile`）。
3. 点击 Inspector 顶部的 **生成（SetupEverything）** 按钮。生成器会：
   - 把模型原地转成 `Physical_Rig`（ArticulationBody 链 + collider，**剥掉渲染**）；
   - 克隆出 `Ghost_Rig`（保留渲染 + Animator + MagicBlend + IK + 三层叠加脚本）；
   - 克隆出 `Shadow_Mimic`（只留 collider，跟随 Ghost）；
   - 自动连线所有隐藏组件，最后 `DestroyImmediate` 掉源 Brain，留下 `{角色名}_RobotRoot`。
4. 进入 Play 模式。**屏幕上可见的是 Ghost_Rig**；Physical 在底下受力但不可见。

> ⚠️ 生成是不可逆的“烘焙”。删组件 ≠ 撤销生成（`ArticulationSyncer` 会在 Play 时自动补回 `CapturePointSolver`）。要改结构请用干净的源模型重新生成。

## 二、三层各自怎么控制

### 第一层 · 基础动画
- 就是普通 Animator。换动作、配 locomotion blend tree 都在 AnimatorController 里做，无额外参数。

### 第二层 · MagicBlend 融合（`globalWeight`）
- 在 Ghost_Rig 的 `MagicBlending` 组件 / `MagicBlendAsset` 上配置叠加/覆盖姿态（武器持握、上半身 overlay 等），遮罩链在编辑器里配（**不要包含 hips**）。
- 强度由 `MagicBlendAsset.globalWeight`（0~1）控制，**由动画侧自己决定**，与物理无关。
- 注意 `MagicBlendAsset` 是 ScriptableObject，多个角色共用同一 asset 会共享 `globalWeight`；多角色需各自复制一份。

### 第三层 · 物理叠加（`balanceUrgency`）
- 由 `BalanceUrgencyEvaluator`（挂 Physical_Rig）输出 `Urgency`(0~1)，`PhysicsPoseBlender`（挂 Ghost_Rig）消费。
- **0 = 纯动画，不介入**；**1 = 把 Physical 当前的整体倾倒按比例叠到可见脊柱链上**。
- 平时 urgency 由「质心加速度偏差 + DCM 发散 + 外部冲击」自动算出：站着没事≈0，被推/踩空→升高。

## 三、编辑器内预览与调试

### 手动钉死 urgency（最常用）
`BalanceUrgencyEvaluator` 上：
- 勾选 **Manual Override**，拖动 **Override Value** 滑杆，即可把 urgency 固定为某值，实时预览第三层混合效果。
- 验证“纯动画”：Override Value = 0 → 角色应**完全不前折**，播放干净动画。
- 验证“物理介入”：滑杆 0→1 → 应看到上半身平滑混入倾倒/自平衡。

### Gizmo / HUD
`RagdollBalanceDebug`（挂 masterRoot）可视化 COM、捕获点 CP、支撑多边形、DCM 箭头，并有 OnGUI 数值。排查平衡问题先看这里。

## 四、调参速查（出问题先调这几个）

| 现象 | 调什么 | 在哪 |
|---|---|---|
| urgency=0 仍然前折 | 确认渲染翻转生效（可见的是 Ghost 不是 Physical）；确认 PhysicsPoseBlender 在 Ghost 上且 chain 已映射 | 生成结果 / `PhysicsPoseBlender` |
| 物理介入时倾倒过猛/抽搐 | `PhysicsPoseBlender.maxLeanAngle`（默认 75°）调小；`blendSmoothTime`（0.06）调大 | `PhysicsPoseBlender` |
| 倾倒分布不自然（头甩太多/胯太僵） | 改 `chainWeights`（默认 hips 重、向上递减），或在 `MapPoseBlenderChain` 改默认权重 | `PhysicsPoseBlender` / `ActiveRagdollBrain` |
| 想要“预判式”朝失衡方向探身 | `pendulumLeanGain` 设 >0（默认 0 = 只用实测倾角） | `PhysicsPoseBlender` |
| urgency 太敏感/太迟钝 | `maxExpectedAccel`（4.9≈0.5g，大=更难触发）、`longFilter`/`shortFilter` | `BalanceUrgencyEvaluator` |
| 被推后不够快抬 urgency | `divergenceWeight`，或外力通道调 `RegisterImpulse` 强度 | `BalanceUrgencyEvaluator` / `CharacterPerimeter` |
| 受击命中点和看到的姿态对不上 | 确认 Shadow 跟随 Ghost（Phase3 已改）；检查 `ShadowRigFollower` 配对 | `ShadowRigFollower` |

> 多数表现参数也在 `RagdollTuningProfile` 里，改 profile 并在 Play 时会经 `ApplyProfile` 实时生效。

## 五、执行顺序（理解时序用）

```
Animator(Fixed pass, 含 MagicBlend Playable jobs)
  → IKSolver(-5)          脚部 IK，锁腿
  → ArmBalanceController(20)   手臂平衡偏移
  → PhysicsPoseBlender(50)     第三层：把物理倾倒叠到脊柱链
  → BalanceUrgencyEvaluator(80) 算 urgency + 物理信号
  → ArticulationSyncer(100)    Physical 关节驱动去追（已叠加修正的）Ghost
LateUpdate: ShadowRigFollower 把 Shadow 贴到 Ghost；CharacterPerimeter 跟踪
```

注意第三层读的是**上一物理帧**的 urgency/倾倒信号（仲裁器 order 80 在它之后），一帧延迟，对混合信号无影响。

---

## 六、关键约束（别踩）

- **腿归 IK**：第三层只动 hips→spine→head，绝不碰腿骨；否则脚会滑出 IK 落点。
- **两层独立**：不要再写“urgency 拉 globalWeight”的桥。
- **自反馈陷阱**：仲裁器读 `PhysicalComVelocity`（未混合），别读 `ComVelocity`（已混合）。
- **asmdef**：ragdoll 脚本在 `FrostPunchGames.asmdef`（references=[]），不能引用 MagicBlend 类型；第三层 `PhysicsPoseBlender` 是路 B 后置叠加、不碰插件类型，所以能留在 asmdef 内。
